# Contributing

Thanks for helping improve Visionary Shield.

## Principles

- Keep security accessible.
- Prefer readable code over clever code.
- Avoid unnecessary permanent threads.
- Keep NUI behavior predictable.
- Document configuration changes.
- Do not include private webhooks, identifiers, or production data.

## Development checklist

Before opening a pull request:

- [ ] Test resource start
- [ ] Test `/zvsadmin`
- [ ] Test dashboard toggle
- [ ] Test text input focus
- [ ] Test player inspector
- [ ] Test screenshot capture with `screenshot-basic`
- [ ] Run basic syntax checks where possible
- [ ] Update docs if behavior changed

## Pull requests

Please describe:

- what changed;
- why it changed;
- how it was tested;
- whether it affects configuration or saved data.
